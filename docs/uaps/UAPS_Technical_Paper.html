Abstract / 초록

UAPS(Undersea Aging Predictive System)는 해저 숙성이 식음료의 풍미에 미치는 영향을 정량적으로 예측하는 2-Layer 하이브리드 AI 시스템이다. 112,316건의 지상 숙성 리뷰 데이터(샴페인, 레드와인, 화이트와인, 사케, 위스키, 콜드브루 커피, 보이차, 간장, 식초, 한국 전통주 10개 카테고리)에서 NLP 기반 6축 풍미 점수를 추출하고, 이를 product_category × aging_stage 그룹으로 클러스터링한 통계 모델(Layer 1)과 Google Gemini AI의 전문가 프로파일 추론(Layer 2)을 앙상블하여 예측한다.

해저 환경의 물리화학적 영향은 아레니우스 방정식 기반 FRI(Flavor Retention Index), 헨리의 법칙 기반 BRI(Bubble Retention Index), 가설적 TCI(Temperature-Pressure Coefficient), 그리고 해류·파고·파주기 기반 K-TCI(Kinetic Texture Coefficient) 네 가지 보정 계수로 모델링한다. 각 계수는 95% 신뢰구간과 과학적 근거를 명시하여 불확실성을 투명하게 관리한다. v4.0에서는 KHOA 국립해양조사원의 실측 데이터(완도 DT_0027 조위관측소 + 완도항 부이 TW_0078)를 14개월간 수집하여 월별 해양 프로파일을 구축하였으며, 표층 수온을 40m 깊이 보정(blending ratio 모델)하여 계절별 FRI/BRI/K-TCI를 동적으로 계산한다.

또한 v4.0은 3단계 비선형 향 감쇠 모델을 도입하여 초기 급감-안정기-후기 급감의 실제 숙성 패턴을 반영하고, 환원취 모델에 수온 의존성을 추가하며, 기존 Conservative Cap(±20점 하드캡)을 개별 물리적 상한(질감/기포 max 95, 향 min 15)으로 대체하였다. 타임라인에는 효율 최적점(추천 인양 시기) 마커를 새로 도입하여 비용 대비 최적 인양 시기를 제안한다.

본 시스템은 10개 식음료 카테고리에 대해 1–36개월 범위의 월별 풍미 변화 타임라인, 골든 윈도우(최적 인양 시기), 종합 품질 점수를 제공하며, AI가 카테고리별 숙성 보정 계수(agingFactors)와 품질 가중치(qualityWeights)를 실시간으로 추론하는 파라미터화된 아키텍처를 채택하였다. 제품 등록 시에는 Gemini 3 Flash + Google Search Grounding을 활용하여 제품 정보를 자동으로 채우는 AI 정보 찾기 기능을 제공한다.

v5.0에서는 네 가지 핵심 확장을 도입하였다. 첫째, Chain-of-Thought 5단계 추론과 카테고리별 Few-shot 전문가 예시를 강제하여 AI 예측의 일관성과 해석 가능성을 높이고, 추론 과정을 ai_reasoning_text 필드에 기록한다. 둘째, XGBoost 기반 ML 모델(Layer 3)을 추가하여 103,314건 학습 데이터에서 6개 풍미 축에 대한 독립적 회귀 모델을 학습하였다. 순수 TypeScript로 JSON 트리를 직접 추론하며 npm 패키지 의존성 없이 동작한다. 셋째, 비교 시음(Comparative Tasting) 시스템을 도입하여 지상 보관(대조군)과 해저 숙성(실험군)의 6축 풍미 점수를 retrieval_results 테이블에 기록하고, 시음 70% + AI 전문가 30% 블렌딩으로 투입 전 프로파일(beforeProfile)의 정확도를 향상시켰다. 넷째, Conjugate Normal-Normal 베이지안 업데이트 엔진으로 인양 실측 데이터가 축적될수록 TCI/FRI/BRI 보정 계수의 불확실성이 자동으로 수렴하는 자기 개선(self-improving) 구조를 구축하였다. 또한 Box-Muller 가우시안 노이즈 기반 시뮬레이션 엔진으로 실제 인양 전 베이지안 엔진의 수렴 패턴을 사전 검증한다.

1. 서론 (Introduction)

해저 숙성(Undersea Aging)은 와인, 증류주 등을 해저 환경에 침지하여 독특한 풍미 변화를 유도하는 기법이다. 해저 환경은 (1) 항온(약 4°C), (2) 차광(완전 암실), (3) 수압(수심 30m 기준 약 4 bar 외부 수압), (4) 미세 진동(해류 및 조류)이라는 네 가지 물리적 조건을 제공하며, 이는 지상 셀러의 12–15°C 항온 환경과 근본적으로 다른 숙성 궤적을 만든다.

그러나 해저 숙성에 대한 체계적인 과학 연구는 매우 제한적이다. 기존의 해저 숙성은 마케팅 기법으로 활용되었을 뿐, 숙성 과정에서 실제로 어떤 화학적·물리적 변화가 일어나며 그것이 풍미에 어떤 영향을 미치는지에 대한 정량적 예측 모델은 존재하지 않았다.

본 논문은 이러한 공백을 메우기 위해 개발된 UAPS(Undersea Aging Predictive System)의 기술적 구조, 과학적 근거, 데이터 파이프라인, 그리고 예측 알고리즘을 상세히 기술한다. 특히 샴페인뿐 아니라 레드와인, 사케, 위스키, 콜드브루 커피, 보이차, 간장, 식초 등 10개 카테고리로 확장된 다중 카테고리 아키텍처와, AI가 카테고리별 보정 계수를 실시간 추론하는 파라미터화 설계를 중점적으로 다룬다.

2.3 숙성 연수 추출 및 신뢰도

리뷰에서 숙성 연수(aging_years)를 추출할 때 3단계 신뢰도 체계를 적용한다:

방법신뢰도예시
빈티지 + 시음 연도 계산0.92"2015 빈티지, 2023 시음" → 8년

신뢰도 0.75 미만인 데이터는 통계 모델 학습에서 제외하여 오염된 학습 데이터 유입을 방지한다.

3. 시스템 아키텍처 (System Architecture)

UAPS는 2-Layer 하이브리드 아키텍처를 채택한다. Layer 1(통계 학습)이 데이터 기반 기준선을 제공하고, Layer 2(AI 추론)가 제품 고유 특성과 카테고리별 보정 계수를 추론하며, 최종 예측은 두 레이어의 앙상블(AI 70% + 통계 30%)로 산출된다.

3.1 예측 파이프라인 개요

전체 파이프라인은 9단계로 구성된다:

  1. 데이터 수집 — 10개 카테고리 112K건 리뷰 수집 (Supabase PostgreSQL)
  2. NLP 6축 추출 — Ollama LLM 배치 처리, 풍미 점수 + 숙성 연수 추출
  3. 지상 시간 변화 추론 — product_category별 연도별 풍미 전이 곡선 생성
  4. Layer 1 통계 학습 — product_category × aging_stage 36개 그룹 클러스터링
  5. XGBoost ML 추론 — 103K건 학습 데이터, 6축 독립 모델, JSON 트리 직접 추론 (npm 의존성 없음) (v5.0)
  6. 해양 환경 데이터 수집 — KHOA 국립해양조사원 11개 API 연동 + 14개월 일별 → 12개월 월별 프로파일 구축
  7. Layer 2 AI 추론 — Gemini AI 전문가 프로파일 + agingFactors/qualityWeights 추론 + 월별 해양 프로파일 컨텍스트
  8. 비교 시음 보정 — 지상 보관(대조군) + 해저 숙성 시음 데이터, 시음 70% + AI 30% 블렌딩 (v5.0)
  9. 해저 환경 보정 — TCI, K-TCI, FRI, BRI 네 가지 보정 계수 적용
  10. 복합 품질 계산 — 가중 합산 + 시너지 보너스 + 골든 윈도우 탐색
  11. 최종 예측 리포트 — 레이더 차트, 타임라인, 인사이트 텍스트 출력
  12. 베이지안 업데이트 — 인양 실측 데이터로 TCI/FRI/BRI 사후분포 업데이트 (v5.0)

3.2 Layer 1: 통계 클러스터링

112K건 데이터를 product_category(10종) × aging_stage(4단계: youthful, developing, mature, aged) 조합으로 최대 40개 그룹(실제 유효 그룹: 약 36개, 최소 5건 이상 필터)으로 분류한다. 각 그룹에서 산출하는 통계량은 다음과 같다:

  • 풍미 프로파일: 6축 각각의 평균, 중앙값, 표준편차, p25, p75
  • 물리화학 통계: pH, dosage, alcohol 분포
  • 풍미 전이 곡선: 해당 카테고리의 연수별 6축 평균 추이
  • 클러스터 중심점: pH × dosage × reduction 조합별 빈도
  • 음용 적기 분포: drinkingWindow start/end 통계
  • 신뢰도: min(sampleCount / 100, 1.0)

유사 클러스터 매칭

예측 시 입력 제품과 가장 유사한 학습 모델 상위 5개를 찾아 가중 평균으로 기준 풍미를 산출한다. 유사도 점수는 다음 4가지 기준의 합산이다:

similarity = categoryMatch(+50) + pHProximity(+20) + dosageProximity(+15) + sampleBonus(+15) (1)

Pseudo-cohort 앵커 방식

단면(cross-sectional) 데이터의 한계를 보완하기 위해, 투하 직전 상태의 클러스터 평균을 t=0 앵커로 고정하고, 목표 숙성 단계와의 변화량(Δ)에만 TCI/FRI 보정을 적용한다. 이 방식은 절대값의 편향을 상쇄하고 변화율만 추출한다.

predicted(t) = anchor_t0 + Δ(target − anchor) × correction(TCI, FRI, months) (2)

3.3 Layer 2: AI 추론 (Gemini)

Google Gemini API(gemini-3-flash-preview → 2.5-flash → 2.5-flash-lite 폴백 체인)를 사용하여 두 가지 핵심 기능을 수행한다:

(a) 전문가 프로파일 생성

제품명과 생산자 정보를 바탕으로 Google Search Grounding을 통해 전문가 테이스팅 노트를 검색하고, 6축 풍미 프로파일을 생성한다. 이는 통계 모델이 제공하는 "같은 종류의 평균"을 넘어 "해당 제품 고유의 특성"을 반영한다.

(b) 카테고리별 보정 계수 추론

AI는 풍미 프로파일과 함께 두 가지 파라미터 세트를 실시간으로 추론한다:

  • agingFactors: 숙성 곡선의 형태를 결정하는 4개 파라미터
    • baseAgingYears: 투하 전 평균 숙성 기간 (0–30년)
    • textureMult: 질감 가속 배수 (0.3–1.5)
    • aromaDecay: 향 감소 속도 배수 (0.3–1.5)
    • riskMult: 환원/결함 리스크 배수 (0.3–2.0)
  • qualityWeights: 복합 품질 점수의 4개 지표 가중치 (합계 = 1.0)
    • texture, aroma, bubble, risk

AI가 계수를 반환하지 못하면 기본값(texture 1.0, aroma 1.0, risk 1.0 / 가중치 0.30·0.30·0.25·0.15)으로 폴백한다. 반환된 값은 범위 클램핑 후, qualityWeights는 합계 1.0으로 자동 정규화된다.

(c) Chain-of-Thought 추론 (v5.0 신규)

v5.0에서는 AI 추론의 일관성과 해석 가능성을 높이기 위해 5단계 Chain-of-Thought(CoT) 추론을 강제한다. 프롬프트에 5단계 분석을 명시적으로 요구하며, AI는 각 단계의 추론 과정을 ai_reasoning_text 필드에 기록한다:

  1. 제품 특성 분석 — 카테고리, 서브타입, pH, dosage, 마개 유형 등 기본 속성 분석
  2. 지상 숙성 상태 평가 — 현재 풍미 프로파일과 숙성 단계 진단
  3. 해양 환경 영향 분석 — 월별 수온, 수압, 해류가 각 풍미 축에 미치는 영향 추론
  4. 시간 경과 예측 — 1~36개월 숙성 궤적과 변곡점 예측
  5. 종합 판단 및 보정 계수 결정 — agingFactors, qualityWeights 최종 결정

또한 카테고리별 전문가 역할(EXPERT_ROLES)을 10개 정의하여 AI가 해당 분야 전문가의 관점에서 추론하도록 유도한다 (예: 샴페인 → WSET Diploma Holder, 사케 → 唎酒師, 위스키 → Master of Malt). 각 카테고리에 대해 Few-shot 예시를 프롬프트에 포함하여 출력 형식과 추론 깊이의 일관성을 확보한다.

3.4 앙상블

최종 풍미 예측은 AI 결과와 통계 결과의 가중 평균이다:

final_flavor(axis) = AI_prediction(axis) × 0.7 + statistical_prediction(axis) × 0.3 (3)

이 앙상블 비율(7:3)은 AI가 제품 고유 특성을 더 잘 포착하되, 통계 모델이 과적합을 방지하는 앵커 역할을 하도록 설계되었다.

4. 해저 환경 보정 계수 (Undersea Correction Coefficients)

해저 환경이 숙성에 미치는 영향을 네 가지 독립적인 보정 계수로 모델링한다. 각 계수는 과학적 근거 수준(가설/문헌/법칙 기반)과 95% 신뢰구간을 명시하여 불확실성을 투명하게 관리한다. v4.0에서는 KHOA 실측 데이터 기반의 월별 해양 프로파일을 사용하여 계절 변동을 동적으로 반영한다 (7절 참조).

4.1 TCI — 온도·압력 계수 (Temperature-Pressure Coefficient)

속성값
기본값0.30
95% 신뢰구간[0.06, 0.54]
근거 수준가설적 추정 (hypothesis)
의미해저에서 질감 성숙이 1/TCI = 약 3.3배 가속 (기본)
최대 가속 제한K-TCI 합산 시 최대 5.0배 (effectiveTci ≥ 0.20)

TCI는 해저의 저온·고압 환경이 효모 자가분해(autolysis)와 질감 발달을 가속시키는 정도를 모델링한다. K-TCI(4.4절)의 운동학적 인자(kf)가 추가 보정되어, 최종 유효 TCI는 다음과 같이 계산된다:

effectiveTci = max(0.20, TCI / kf) (4a)

여기서 0.20 하한 클램핑은 최대 가속 배수가 1/0.20 = 5.0배를 초과하지 않도록 제한한다. K-TCI 도입 이전에는 TCI 단독으로 3.3배 가속이었으나, 운동학적 인자까지 합산하면 비현실적으로 높은 가속(6~7배)이 도출될 수 있어 보수적 상한을 설정하였다.

질감 성숙도는 시그모이드 함수로 계산된다:

textureMaturity = 100 / (1 + exp(−0.6 × (baseYears + months/(12 × effectiveTci × textureMult) − 4.5))) (4b)

중점(inflection point)은 4.5년, 기울기 0.6으로 설정하여 36개월 전 구간에서 점진적 변화를 유지한다. textureMult는 AI가 추론한 카테고리별 질감 가속 배수이다.

주의: TCI는 직접적인 선행 연구가 없어 넓은 신뢰구간을 가진다. 인양 후 블라인드 시음 데이터 축적에 따라 베이지안 업데이트로 수렴시킬 계획이다. 최대 5배 제한은 현재까지의 해양 숙성 문헌 범위(1.5~4배)를 고려한 보수적 상한이다.

4.2 FRI — 향 신선도 지수 (Flavor Retention Index)

속성값
기본값0.565
95% 신뢰구간[0.524, 0.608]
근거 수준아레니우스 방정식 (Arrhenius equation)
의미해저에서 산화 속도가 지상의 56.5% 수준으로 감속

FRI는 아레니우스 방정식으로 도출된다. 화학 반응 속도와 온도의 관계:

k(T) = A × exp(−Ea / RT) (5)

두 온도에서의 속도 비:

FRI = k(Tocean) / k(Tcellar) = exp[(−Ea/R) × (1/Tocean − 1/Tcellar)] (6)

적용 파라미터:

  • 해저 온도 Tocean = 4°C = 277.15 K
  • 지상 셀러 온도 Tcellar = 12°C = 285.15 K
  • 기체 상수 R = 8.314 J/(mol·K)
  • 활성화 에너지 Ea = 카테고리별 개별 적용 (아래 테이블 참조)

4.2.1 카테고리별 활성화 에너지 (Ea) 참조 테이블

4.2.2 마개(Closure) 산소 투과율 보정

아레니우스 방정식만으로는 마개를 통한 산소 유입(Oxygen Ingress)의 영향을 반영할 수 없다. 해저에서는 수압이 높아 달턴의 법칙에 의해 코르크 기공의 산소 투과 구동력이 변하므로, Closure Factor를 FRI에 곱하는 보정을 도입한다:

FRIadjusted = FRIarrhenius × (1 + OTRadjusted / 10) (6a)
OTRadjusted = OTRbase × max(0.2, 1 − depth / 60) (6b)

여기서 OTRbase는 마개 유형별 기본 산소 투과율(mg O₂/year), 수심 보정은 60m 기준으로 선형 감소하며 최소 20%로 클램핑한다. 수압이 코르크 기공을 물리적으로 압축하여 투과율을 낮추는 효과를 반영한다.

향 신선도는 v4.0에서 3단계 비선형 감쇠 모델로 교체되었다. 기존의 단순 지수 감쇠(−0.10 × years)는 초기 신선 과일향 손실, 숙성향 전환기, 후기 향 피로를 구분하지 못하는 한계가 있었다. 새 모델은 실제 숙성 패턴에 가까운 3단계 곡선을 적용한다:

t = totalOxidationYears = baseYears + months × FRIadjusted × aromaDecay / 12 (7a)
단계 1 (0 ≤ t < 2년): score = 100 × exp(−0.15 × t) — 급감 (신선 과일향 손실) (7b)
단계 2 (2 ≤ t < 3.5년): score = phase1End × exp(−0.05 × (t − 2)) — 안정 (숙성향 전환) (7c)
단계 3 (t ≥ 3.5년): score = phase2End × exp(−0.12 × (t − 3.5)) — 후기 급감 (향 피로) (7d)

여기서 phase1End = 100 × exp(−0.15 × 2) ≈ 74.1, phase2End = phase1End × exp(−0.05 × 1.5) ≈ 68.7이다. 각 단계의 경계에서 연속적으로 이어지므로 점프 없는 매끄러운 곡선을 형성한다.

4.3 BRI — 기포 안정화 지수 (Bubble Retention Index)

속성값
기본값1.60
95% 신뢰구간[1.37, 1.92]
근거 수준헨리의 법칙 (Henry's Law)
의미해저에서 기포가 지상 대비 1.6배 더 안정적으로 보존

BRI는 헨리의 법칙에 기반한 CO₂ 손실 구동력 비교로 도출된다. 샴페인 내부 압력을 6 bar로 가정할 때:

BRI = (Pland_driving / Pocean_driving) × kH_ratio (8)

구체적 계산:

  • 지상 구동력: Pbottle − Patm = 6 − 1 = 5 bar
  • 해저 구동력 (수심 30m): Pbottle − (1 + 30/10) = 6 − 4 = 2 bar
  • 압력 비: 5 / 2 = 2.5
  • CO₂ 용해도 온도 보정: kH(4°C) / kH(12°C) ≈ 1 + 0.03 × (12 − 4) = 1.24
  • BRI 원시값 = 2.5 × 1.24 ≈ 3.10 → 실효값 1.60 (보정)

기포 미세화는 로그 성장 곡선으로 모델링된다:

bubbleRefinement = 40 + 60 × (1 − exp(−rate × months)) (9)
rate = 0.08 × (BRI / 1.6) × min(depth / 30, 1.5) (10)

4.4 K-TCI — 운동학적 질감 계수 (Kinetic Texture Coefficient)

속성값
기본값동적 (해양 환경에 따라 변동)
근거 수준해양 역학 — 궤도 에너지 밀도
의미해류·파고·파주기가 질감 발달에 미치는 운동학적 효과

K-TCI는 해저의 운동학적 에너지가 질감 발달에 미치는 추가 보정을 모델링한다. 해류속도, 파고, 파주기를 입력으로 받아 동적 보정 계수(kf)를 산출하며, 이 값이 TCI의 질감 성숙도 계산에 가산된다.

kf = f(currentVelocity, waveHeight, wavePeriod) (10a)

파주기(wave period)에 의한 보정은 궤도 에너지 밀도 관계에 기반한다:

Eorbital ∝ H² / T (10b)

구체적 보정 규칙:

파주기 범위해석kf 보정
< 5초풍파(wind wave) — 고주파 진동, 높은 에너지 밀도+0.15
5–8초보통 주기 — 표준 에너지+0.05
8–12초정상 범위 — 기본 수준±0
> 12초너울(swell) — 완만한 움직임, 낮은 에너지−0.05

해류속도 보정은 0.1 m/s 이상일 때 kf에 +0.1을 가산하고, 파고 보정은 0.5m 이상일 때 +0.1, 1.5m 이상일 때 +0.2을 가산한다. 최종 kf는 [0, 1.0] 범위로 클램핑된다.

데이터 소스 (v4.0 업데이트): v3.1까지는 Open-Meteo Marine API의 전체 기간 통계 평균을 사용하였으나, v4.0부터 KHOA 국립해양조사원의 실측 데이터(완도 DT_0027, 완도항 TW_0078)로 전환하였다. 14개월간 일별 수집된 데이터를 12개월 월별 프로파일로 집계하여 계절별 해양 환경 변동을 동적으로 반영한다 (7절 참조). Open-Meteo 데이터는 KHOA 미지원 기간의 백필 및 교차 검증 용도로 유지한다.

5. 복합 품질 점수 및 골든 윈도우 (Composite Quality & Golden Window)

5.1 복합 품질 점수

4개 지표(질감 성숙도, 향 신선도, 기포 미세화, 환원취 리스크)를 AI가 추론한 카테고리별 가중치(qualityWeights)로 합산한다:

compositeQuality = Σ(scorei × weighti) + synergyBonus (11)

여기서 risk 지표는 100 − offFlavorRisk로 역전하여 높을수록 좋은 방향으로 통일한다.

개별 물리적 상한 (v4.0 변경)

v3.1까지는 compositeQuality에 ±20점의 Conservative Cap(하드캡)을 적용하여 통계 모델 대비 과도한 이탈을 억제하였다. 그러나 이 방식은 카테고리 간 특성 차이를 무시하고 품질 점수의 표현 범위를 인위적으로 제한하는 문제가 있었다.

v4.0에서는 종합 캡을 제거하고, 개별 지표에 물리적 근거가 있는 상한/하한을 적용한다:

지표상한하한근거
질감 성숙도95—완벽한 자가분해는 현실적으로 불가
기포 미세화95—물리적 이상 상태(무한 미세화) 방지
향 신선도—15잔여 3차 향(미네랄, 흙)은 완전 소멸하지 않음
환원취 리스크95—100% 결함은 물리적으로 불가

시너지 보너스

4개 지표가 고르게 높을 때 추가 보너스가 발동된다:

  • 균형도(harmony) = min(scores) / max(scores)
  • 평균 ≥ 60점 이고 harmony ≥ 0.5일 때 활성화
  • 최대 +15점: 15 × avgFactor × harmonyFactor

5.3.1 수온 의존성 (v4.0 신규)

v4.0에서는 환원취 발생 속도에 해수 온도 의존성을 도입하였다. 기존 모델은 환원취를 오직 시간의 함수로만 계산했으나, 혐기 환경에서의 황 화합물 생성 반응은 온도에 의존하므로 수온 보정이 필요하다:

tempFactor = 1 + (seaTemperature − 10) × 0.03 (12a)
riskadjusted = risk × tempFactor (12b)

기준 온도 10°C 대비, 여름 고온(예: 18°C)에서는 tempFactor = 1.24로 환원취 발생이 최대 +24% 가속되며, 겨울 저수온(예: 4°C)에서는 tempFactor = 0.82로 최대 −18% 억제된다. KHOA 월별 해양 프로파일에서 해당 월의 해저 수온을 읽어 동적으로 적용한다.

6. 다중 카테고리 확장 아키텍처 (Multi-Category Architecture)

6.1 카테고리 분기 설계

10개 카테고리는 물리화학적 특성이 근본적으로 다르므로, 단일 모델로 모든 카테고리를 다루는 것은 부적절하다. UAPS는 다음 세 가지 수준에서 카테고리별 분기를 구현한다:

  1. 폼 분기: 제품 등록 시 카테고리에 따라 표시 필드를 동적 변경 (예: 샴페인만 dosage 표시, 비주류는 alcohol 숨김)
  2. 환원 체크리스트 분기: 카테고리별 고유 체크리스트 (예: 위스키 피트 레벨, 사케 생주 여부, 커피 로스트 레벨)
  3. 보정 계수 분기: AI가 agingFactors/qualityWeights를 카테고리별로 실시간 추론

6.2 환원 성향 자동 산출

기존의 수동 low/medium/high 선택 대신, 카테고리별 체크리스트의 가중 합산으로 자동 산출한다. 각 항목은 가중치(weight)와 그룹(group)을 가지며, 같은 그룹 내 항목은 라디오 선택(하나만), 비그룹 항목은 복수 선택이 가능하다.

reductionScore = Σ(checked_item.weight) (13)
reductionPotential = score ≥ 3 → 'high' | score ≥ 1 → 'medium' | else → 'low' (14)

7.2 깊이 보정 (SST → 해저 수온)

KHOA 관측소는 표층 수온(SST, Sea Surface Temperature)을 측정한다. 해저 숙성 환경(목표 수심 약 40m)의 실제 수온을 추정하기 위해 blending ratio 모델을 적용한다:

bottomTemp = SST × ratio + DEEP_BASE × (1 − ratio) (15)

여기서 DEEP_BASE = 8°C는 한국 남해 심층 기저 온도(연중 안정), ratio는 월별로 변하는 표층-저층 연동 비율이다.

10. 베이지안 계수 업데이트 (v5.0 신규)

10.1 Conjugate Normal-Normal 모델

UAPS의 핵심 불확실성인 보정 계수(TCI, FRI, BRI)를 인양 실측 데이터로 베이지안 업데이트하는 엔진을 구현하였다. Conjugate Normal-Normal 모델을 사용하므로 MCMC 없이 해석적 닫힌 형태(closed-form)로 업데이트된다.

사전분포: μ ~ N(μ₀, σ₀²) (18a)
우도: x | μ ~ N(μ, σ_obs²) (18b)
사후분포: μ₁ = (μ₀/σ₀² + n·x̄/σ_obs²) / (1/σ₀² + n/σ_obs²) (18c)
σ₁² = 1 / (1/σ₀² + n/σ_obs²) (18d)
Table 10. 보정 계수 사전분포
계수사전 평균 (μ₀)사전 분산 (σ₀²)관측 분산 (σ_obs²)
TCI0.400.040.05
FRI0.560.020.05
BRI0.720.020.05

관측 데이터가 축적될수록 사후 분산이 줄어들어 계수 추정의 확실성이 증가한다. 분산 축소율(convergenceRate = 1 − σ₁²/σ₀²)이 모니터링되며, 10건 이상의 인양 데이터에서 약 50% 이상의 분산 축소가 예상된다.

10.2 시뮬레이션 엔진

실제 인양 데이터가 충분하지 않은 현 단계에서 베이지안 엔진을 검증하기 위해 가상 인양 데이터 생성기(Simulation Engine)를 구현하였다. Box-Muller 변환으로 정규분포 노이즈(기본 ±10%)를 예측값에 추가하여 가상 실측 데이터를 생성하고, 이를 베이지안 엔진에 공급하여 계수 수렴 패턴을 사전 검증한다.

11.2 현존하는 한계

  1. TCI 미검증: 95% CI [0.06, 0.54]로 불확실성이 매우 크다. 실제 인양 후 블라인드 시음 데이터로만 검증 가능하며, 베이지안 업데이트를 계획하고 있다.
  2. Survivorship Bias: 오래된 와인 리뷰는 품질이 뛰어나 살아남은 것만 남아, 고령 구간 클러스터의 품질이 과대평가될 수 있다.
  3. 카테고리별 보정 계수 실측 검증 필요: AI가 추론하는 agingFactors/qualityWeights는 학습 데이터 기반 추정이며, 각 카테고리의 실제 해저 숙성 결과로 검증이 필요하다.
  4. 비주류 카테고리 데이터 불균형: 식초(702건), 위스키(435건) 등은 통계적 신뢰도가 제한적이다. 추가 데이터 수집 및 외부 데이터셋 확보가 필요하다.

11.3 향후 로드맵

  • 2026년 하반기: 첫 인양 → 블라인드 시음 → TCI 베이지안 업데이트
  • 인양 데이터 축적 시: 예측 vs 실측 비교 → 모델 정확도 평가 및 리캘리브레이션
  • 과학적 타당성 점수: 현재 6.2/10 → 인양 후 7.5+/10 목표
  • v5.0 즉시: 가상 시뮬레이션으로 베이지안 엔진 사전 검증 → 수렴 패턴 확인

12. 결론 (Conclusion)

UAPS는 해저 숙성이라는 미개척 영역에 대해 체계적인 정량 예측 프레임워크를 제시하였다. 112,316건의 다중 카테고리 지상 숙성 데이터를 기반으로, 아레니우스 방정식(FRI)과 헨리의 법칙(BRI)이라는 확립된 물리화학 법칙을 적용하고, 해양 역학 기반 K-TCI(운동학적 질감 보정)를 도입하여 파주기·파고·해류속도의 영향을 정량화하였으며, 통계 클러스터링과 AI 추론의 앙상블로 예측 정밀도를 높였다.

v4.0에서는 세 가지 핵심 개선을 도입하였다. 첫째, KHOA 국립해양조사원의 실측 데이터(14개월, 완도 해역)를 통합하고 깊이 보정(blending ratio)을 적용하여 계절별 해양 환경 변동을 월 단위로 동적 반영한다. 둘째, 3단계 비선형 향 감쇠 모델로 초기 급감-안정기-후기 급감의 실제 숙성 패턴을 포착하고, 환원취에 수온 의존성을 추가하여 여름/겨울 계절 효과를 반영한다. 셋째, Conservative Cap을 제거하고 개별 물리적 상한을 도입하며, 효율 최적점(Peak − 4점 이내 첫 도달) 마커로 비용 대비 최적 인양 시기를 제안한다.

v5.0에서는 네 가지 핵심 확장을 도입하였다. 첫째, Chain-of-Thought 5단계 추론과 카테고리별 전문가 역할로 AI 예측의 일관성과 해석 가능성을 높였다. 둘째, XGBoost ML을 제3의 레이어로 추가하여 103K건 학습 데이터 기반의 독립적 교차 검증을 가능하게 하였다. 셋째, 비교 시음 시스템으로 지상 대조군과 해저 실험군의 정량적 비교를 표준화하고, 시음 70% + AI 30% 블렌딩으로 투입 전 프로파일의 정확도를 향상시켰다. 넷째, Conjugate Normal-Normal 베이지안 업데이트 엔진으로 인양 실측 데이터가 축적될수록 TCI/FRI/BRI 보정 계수의 불확실성이 자동으로 수렴하는 자기 개선(self-improving) 구조를 구축하였다.

AI가 카테고리별 보정 계수(agingFactors, qualityWeights)를 실시간 추론하는 파라미터화된 아키텍처는 샴페인뿐 아니라 커피, 사케, 보이차, 간장 등 근본적으로 다른 식음료에도 동일한 프레임워크를 적용할 수 있게 하였다. 제품 등록 시 AI 정보 찾기(Gemini + Google Search Grounding)를 통해 사용자 입력 부담을 줄이고 데이터 정확도를 높였다.

현재 가장 큰 불확실성인 TCI(질감 가속 계수)는 첫 인양 후 실측 데이터로 검증될 예정이며, 이를 통해 모델의 과학적 타당성이 크게 향상될 것으로 기대한다.
