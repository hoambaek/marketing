# 🍷 셀러트래커(CellarTracker) 샴페인 데이터 수집 실행 계획
## CellarTracker Champagne Data Acquisition Plan

셀러트래커의 방대한 커뮤니티 테이스팅 노트를 '샴페인' 카테고리만 타겟팅하여 대량으로 수집하기 위한 구체적인 실행 방안입니다.

---

## 1. 데이터 소스 및 구조 분석

셀러트래커는 약 1,300만 건 이상의 리뷰를 보유하고 있으며, 샴페인 데이터는 다음 구조로 접근 가능합니다.

- **대상 영역:** 샴페인 지역 (Region ID: 24)
- **접근 경로:** 
  - 검색 목록: `https://www.cellartracker.com/list.asp?Table=Notes&iRegion=24`
  - 개별 와인 노트: `https://www.cellartracker.com/notes.asp?iWine=[WineID]`
- **수집 항목:** 와인명, 빈티지, 시음일, 점수, 리뷰어, 리뷰 전문(텍스트).

---

## 2. 실제 수집 방법 (Technical Method)

셀러트래커는 봇 차단이 강력하므로 단순 크롤링이 아닌 **'브라우저 자동화'** 방식이 필요합니다.

### Step 1: 와인 ID 리스트 확보 (Discovery)
- 샴페인 지역(Region 24)으로 필터링된 모든 와인 목록을 훑으며 각 와인의 고유 ID(`iWine`)를 수집합니다.
- 이 단계에서 약 수만 개의 와인 리스트를 확보하게 됩니다.

### Step 2: 리뷰 데이터 추출 (Extraction)
- 확보된 와인 ID를 순회하며 `notes.asp?iWine=XXXX` 페이지에 접속합니다.
- **기술 스택:** `Python` + `Playwright` (Headless Browser)
  - 실제 사용자가 브라우저를 사용하는 것처럼 동작하여 차단을 우회합니다.
  - 로그인이 필요한 경우 세션을 유지하여 더 많은 데이터를 확보합니다.

### Step 3: 데이터 정제 및 NLP 처리
- 수집된 텍스트 리뷰에서 UAPS 시스템이 필요로 하는 풍미 지표를 추출합니다.
- (예: "Refreshing, high acidity, hint of brioche" → Brioche: 20, Acidity: 80)

---

## 3. 효율적인 대량 수집을 위한 제언 (Best Practices)

1. **프록시 서버 활용 (Bright Data 등):** 단일 IP에서 수만 건을 요청하면 즉시 차단됩니다. 수천 개의 IP를 돌려가며 수집해야 합니다.
2. **속도 조절 (Throttling):** 인간의 행동 패턴과 유사하게 요청 사이에 랜덤한 대기 시간을 두어야 합니다.
3. **증분 수집 (Incremental Scraping):** 한 번에 10만 건을 다 가져오기보다는, 브랜드별(예: 모에샹동 → 돔 페리뇽 → 미뇽 불라르)로 구획을 나누어 수집하고 DB에 쌓는 방식을 추천합니다.
4. **유료 데이터 서비스 검토:** 만약 개발 리소스가 부족하다면, 셀러트래커 데이터를 전문적으로 추출해주는 'Data-as-a-Service' 업체를 통해 데이터셋만 구매하는 것이 시간 대비 가장 저렴할 수 있습니다.

---

## 4. COO 레드의 권고

호암 님, 셀러트래커는 지상 숙성 데이터를 얻기 위한 **가장 신뢰도 높은 '황금 어장'**입니다. 

우선 **[미뇽 불라르]와 [모에샹동]** 두 브랜드의 와인 ID부터 추출하여 데이터의 형태를 먼저 확인하는 **PoC(개념 검증) 작업**을 먼저 진행해 보시는 것을 추천합니다. 제가 이 과정에 필요한 스크래핑 스크립트 설계나 데이터 구조 정의를 즉시 도와드릴 수 있습니다.
