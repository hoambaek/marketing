# UAPS v5.0 AI 고도화 구현 플랜

> 100사이클 Autoresearch 결과 기반 — 14 Keep 가설의 우선순위별 구현 계획

**Goal:** UAPS를 텍스트 기반 예측에서 과학적 디지털 트윈 시스템으로 발전. 빠른 승리(3개월) → 중기(6개월) → 장기(인양 후)

---

## Phase 1: 빠른 승리 (1~3개월) — 현재 인프라로 즉시 가능

### 1-1. What-if 시뮬레이터 [H37, 72점]
**효과:** 사용자가 "수심 50m→30m면?" 즉시 비교 가능

**구현:**
- UAPS 페이지에 조건 슬라이더 UI 추가 (수심, 기간, 마개, 투입월)
- `generateTimelineData()`에 파라미터만 바꿔서 즉시 재계산
- 두 타임라인을 오버레이로 비교 표시

**파일:** `src/app/uaps/page.tsx` (UI), `src/lib/utils/uaps-engine.ts` (기존 함수 재활용)
**난이도:** 하 | **기간:** 1~2주

---

### 1-2. 목표 품질 역산 [H31, 74점]
**효과:** "종합 85점 이상을 원하면 몇 개월?" 자동 응답

**구현:**
- `generateTimelineData()` 결과에서 목표 점수 이상인 첫 시점 탐색
- 역함수가 아닌 **순방향 탐색 + 이진 검색**: 1~36개월 중 목표 도달 최초 시점
- UAPS 대시보드에 "목표 품질" 입력 → 추천 기간 표시

**파일:** `src/lib/utils/uaps-engine.ts` (함수 추가), `src/app/uaps/page.tsx` (UI)
**난이도:** 하 | **기간:** 1주

---

### 1-3. Gemini 프롬프트 체계적 최적화 [H69, 71점]
**효과:** AI 예측 정확도 + 일관성 향상

**구현:**
- 현재 수동 작성된 프롬프트를 **템플릿 + 변수** 구조로 분리
- Chain-of-Thought 추론 강제: "단계별로 분석하세요: 1) 제품 특성 → 2) 해양 환경 → 3) 숙성 예측"
- Few-shot 예시 2~3개 프롬프트에 포함 (잘 예측된 기존 결과)
- 프롬프트 A/B 테스트: 5개 제품에 대해 2가지 프롬프트 비교

**파일:** `src/lib/utils/uaps-ai-predictor.ts`
**난이도:** 중 | **기간:** 2~3주

---

### 1-4. 엔진 단위 테스트 자동화 [H91, 71점]
**효과:** 코드 변경 시 회귀 방지, 모델 신뢰도 증가

**구현:**
- Vitest로 각 엔진 함수 단위 테스트
- 스냅샷 테스트: 특정 제품의 타임라인 결과가 변경되면 알림
- 경계값 테스트: 극단 입력 (수온 0°C, 수심 100m, 36개월 등)
- CI에 통합하여 커밋마다 자동 실행

**파일:** `tests/uaps-engine.test.ts` (신규)
**난이도:** 중 | **기간:** 2주

---

### 1-5. 데이터 품질 대시보드 [H71, 70점]
**효과:** 112K 데이터의 건강 상태 한눈에 파악

**구현:**
- 6대 메트릭: 완전성(NULL 비율), 분포 균형, 이상치, 카테고리별 건수
- Supabase SQL로 집계 → API 라우트 → 대시보드 카드
- UAPS 페이지 또는 별도 `/uaps/data-health` 페이지

**파일:** `src/app/api/uaps/data-health/route.ts` (신규), UI
**난이도:** 중 | **기간:** 2주

---

### 1-6. 예측 불확실성 정량화 [H43, 70점]
**효과:** "이 예측은 ±8점 범위" 같은 신뢰 구간 제공

**구현:**
- Monte Carlo 시뮬레이션: FRI/BRI/K-TCI에 ±10% 노이즈 → 100회 반복 → 분포 산출
- 타임라인 차트에 **신뢰 밴드** (95% 구간) 표시
- compositeQuality ± stddev를 밴드로 렌더링

**파일:** `src/lib/utils/uaps-engine.ts` (MC 함수), `src/app/uaps/page.tsx` (밴드 UI)
**난이도:** 중 | **기간:** 3주

---

### 1-7. 가중 손실 함수로 카테고리 불균형 해소 [H57, 70점]
**효과:** 소수 카테고리(위스키 435건) 예측 정확도 향상

**구현:**
- 클러스터링 시 가중치 = 1/카테고리 빈도
- 샴페인(40K) 가중치 1x vs 위스키(435) 가중치 ~93x
- `retrain_model.mjs`에 가중 평균 적용

**파일:** `data/scripts/retrain_model.mjs`
**난이도:** 하 | **기간:** 1주

---

## Phase 2: 중기 (3~6개월) — 인프라 확장 필요

### 2-1. Layer 1 ML 전환 — XGBoost [H93, 72점]
**효과:** 클러스터 평균 → ML 모델로 예측 정확도 대폭 향상

**구현:**
- 112K 데이터로 XGBoost 학습 (Python)
- 입력: category, aging_stage, pH, dosage, alcohol 등
- 출력: 6축 풍미 점수
- ONNX로 변환 → Next.js 서버사이드 추론 or API 분리

**의존성:** Python ML 파이프라인 구축 필요
**난이도:** 상 | **기간:** 1~2개월

---

### 2-2. Layer 2 Gemini에 ML 예측 주입 [H94, 72점]
**효과:** AI가 ML 모델의 예측+불확실성을 컨텍스트로 활용

**구현:**
- Phase 2-1의 XGBoost 예측 결과를 Gemini 프롬프트에 포함
- "ML 모델 예측: 과실향 65±8, 질감 72±5 → 이를 참고하여 전문가 판단"
- ML + LLM 앙상블 가중 평균

**의존성:** Phase 2-1 완료 후
**난이도:** 중 | **기간:** 2~3주

---

### 2-3. 전이 학습 — 유사 카테고리 지식 전이 [H23, 72점]
**효과:** 위스키(435건), 식초(702건) 등 소수 카테고리 예측 강화

**구현:**
- 레드와인(22K) 산화 모델 → 위스키 적용 (공통 Feature: alcohol, pH, 산화 속도)
- 도메인 적응: 공통 레이어 + 카테고리별 헤드
- Phase 2-1의 XGBoost에 자연스럽게 통합 (피처 공유)

**의존성:** Phase 2-1 완료 후
**난이도:** 상 | **기간:** 3~4주

---

### 2-4. Conformal Prediction [H45, 72점]
**효과:** 모델 무관, 유한 샘플 보장의 예측 구간

**구현:**
- Phase 2-1 ML 모델 + 보정 세트로 nonconformity score 계산
- 새 예측에 대해 보장된 95% 구간 제공
- Phase 1-6(Monte Carlo)보다 이론적으로 강건

**의존성:** Phase 2-1 완료 후
**난이도:** 중 | **기간:** 2주

---

## Phase 3: 장기 (인양 후, 6개월+) — 실측 데이터 필요

### 3-1. 베이지안 TCI 업데이트 [H11, 70점]
**효과:** TCI 95% CI [0.06, 0.54] → 인양 데이터로 좁히기

**구현:**
- 인양 → 블라인드 시음 → 6축 풍미 실측값
- 예측 vs 실측 비교 → 사후분포 계산
- 자동 리캘리브레이션 파이프라인

**의존성:** 첫 인양 (2026년 하반기 예정)
**난이도:** 상 | **기간:** 인양 후 1~2개월

---

### 3-2. 소수 인양 데이터 베이지안 [H14, 72점]
**효과:** N<10에서도 유의미한 모델 업데이트

**구현:**
- 정보적 사전분포(informative prior) 설정: 현재 FRI/BRI 추정값 활용
- Conjugate prior로 해석적 업데이트 (MCMC 불필요)
- 인양 데이터 1건당 사후분포 변화 시각화

**의존성:** Phase 3-1과 병행
**난이도:** 중 | **기간:** 2~3주

---

### 3-3. 시뮬레이션 기반 검증 SBC [H20, 74점]
**효과:** 베이지안 모델이 올바르게 작동하는지 검증

**구현:**
- Simulation-Based Calibration: 사전분포에서 파라미터 샘플링 → 가상 데이터 생성 → 사후분포 회복 확인
- 인양 전에도 가상 데이터로 파이프라인 검증 가능

**의존성:** Phase 3-1 파이프라인 구축 후
**난이도:** 상 | **기간:** 3~4주

---

## 구현 순서 요약

```
Phase 1 (1~3개월, 7건)
├── 1-1 What-if 시뮬레이터 ← 즉시 시작
├── 1-2 목표 품질 역산 ← 즉시 시작
├── 1-3 프롬프트 최적화 ← 즉시 시작
├── 1-4 단위 테스트 ← 즉시 시작
├── 1-5 데이터 품질 대시보드
├── 1-6 불확실성 정량화 (MC)
└── 1-7 가중 손실 함수

Phase 2 (3~6개월, 4건)
├── 2-1 XGBoost ML 전환 ← Phase 1 완료 후
├── 2-2 Gemini + ML 앙상블 ← 2-1 후
├── 2-3 전이 학습 ← 2-1 후
└── 2-4 Conformal Prediction ← 2-1 후

Phase 3 (인양 후, 3건)
├── 3-1 베이지안 TCI 업데이트 ← 인양 후
├── 3-2 소수 데이터 베이지안 ← 3-1과 병행
└── 3-3 SBC 검증 ← 3-1 후
```

## 실행하지 않는 것 (Discard/Watch)

| 방향 | 사유 |
|------|------|
| **GC-MS 이화학 데이터** | 데이터 확보 비현실적, 구현 복잡도 극높음. 논문 참고용으로만 유지 |
| **SMOTE/ADASYN 오버샘플링** | 가중 손실이 더 간단하고 효과적 |
| **생존 편향 수학적 교정** | IPW/역확률 가중치는 생존율 데이터 자체가 없어 적용 불가 |
| **유전 알고리즘 최적화** | What-if 시뮬레이터가 더 직관적이고 구현 간단 |
